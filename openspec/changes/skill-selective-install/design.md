## Context

当前 `install.py` 是一个硬编码单一 skill 安装的脚本。它通过 `TOOL_CONFIGS` 字典定义工具配置，`SKILL_NAME` 常量固定为 `"generate-prd-from-code"`。安装流程为：
1. 选择工具 (Claude Code / Cursor / 全部)
2. 确认安装
3. 执行安装

用户需要选择性地安装特定 skills，但目前脚本不支持此功能。

## Goals / Non-Goals

**Goals:**
- 自动发现 `skills/` 目录下所有可用的 skill 包
- 提供交互式多选界面让用户选择要安装的 skills
- 检测已安装的 skills 并支持选择性卸载
- 支持命令行参数指定 skills 和操作（非交互模式）
- 保持向后兼容，现有使用方式不受影响

**Non-Goals:**
- 不实现 skill 版本管理
- 不实现 skill 依赖解析
- 不实现 skill 在线下载（仅支持本地 skills 目录）

## Decisions

### D1: Skill 发现机制

**决策**: 扫描 `skills/` 目录下的子目录，通过检查 `SKILL.md` 文件存在性来识别有效 skill。

**理由**:
- 简单可靠，与现有 skill 结构一致
- 无需额外配置文件
- 易于扩展新 skill

**备选方案**:
- 使用配置文件注册 skill：增加维护负担
- 扫描 `platforms/` 目录：逻辑更复杂，需要建立 skill-platform 映射

### D2: 交互式选择界面

**决策**: 使用 `questionary` 库实现空格切换选中、上下箭头移动、回车确认的多选界面。

**理由**:
- 现代化的用户体验，符合主流 CLI 工具交互习惯
- `questionary` 是纯 Python 实现，无需编译依赖
- 跨平台兼容（Windows/macOS/Linux）
- API 简洁，易于维护

**交互方式**:
- `↑/↓` 或 `j/k` 移动光标
- `Space` 切换选中状态
- `Enter` 确认选择
- `Ctrl+C` 取消退出

**依赖管理**:
- 优先使用已安装的 `questionary`
- 未安装时提供 `pip install questionary` 提示，或降级为简单输入模式

**备选方案**:
- 使用数字序号输入 `1,2,3`：实现简单但用户体验较差
- 使用 `inquirer` 库：功能类似但更新较少

### D3: 命令行参数支持

**决策**: 支持 `--skills skill1,skill2` 参数指定 skills。

**理由**:
- 支持 CI/CD 自动化安装
- 方便脚本化批量部署
- 与现有交互模式并存，互不干扰

### D4: 安装流程顺序

**决策**: 先选择 skills，再选择工具。

**理由**:
- 逻辑更自然：用户先确定"装什么"，再确定"装到哪"
- 避免重复：多个 skills 安装到同一工具时无需重复选择

### D5: 卸载功能

**决策**: 提供 `--uninstall` 参数进入卸载模式，检测已安装的 skills 并支持选择性删除。

**理由**:
- 用户需要清理不再使用的 skills
- 保持环境整洁，避免占用过多磁盘空间
- 与安装流程对称，用户体验一致

**实现方式**:
- 扫描工具的 skill 目录（如 `~/.claude/skills/`、`~/.cursor/skills/`）
- 显示已安装的 skills 列表，使用 checkbox 多选
- 确认后删除选中的 skill 目录

**交互流程**:
1. 选择工具（或通过 `--tools` 参数指定）
2. 显示该工具下已安装的 skills 列表
3. 用户多选要卸载的 skills
4. 确认后执行删除

**安全措施**:
- 卸载前显示将要删除的目录路径
- 要求用户二次确认
- 单个 skill 时直接确认即可

### D6: 主菜单模式选择

**决策**: 运行脚本时首先显示主菜单，让用户选择"安装"或"卸载"操作。

**理由**:
- 统一入口，避免记忆多个命令
- 与命令行参数 `--uninstall` 共存，两种方式都支持

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| skills 目录为空时无内容可选 | 显示友好提示，引导用户添加 skills |
| 用户输入无效 skill 名称 | 验证输入，显示可用列表 |
| 大量 skills 时列表过长 | 考虑分页或搜索（未来迭代）|
| 命令行参数格式错误 | 提供清晰的错误提示和使用示例 |
| 误删重要 skill | 卸载前二次确认，显示删除路径 |
| questionary 未安装 | 降级为简单输入模式，提示用户可安装 questionary 获得更好体验 |
