## 新增需求

### 需求: Cursor Rules 文件生成

系统应提供 `.cursorrules` 文件，定义在 Cursor IDE 中从代码生成 PRD 的完整工作流。

#### 场景: 用户以纯代码模式执行 generate-prd-from-code
- **当** 用户运行 skill 且未提供前端 URL
- **则** 系统依次执行 `detect_stack.py`、`extract_repo_facts.py` 和 `merge_evidence.py` 脚本
- **且** 在 `docs/prd/` 目录下生成 AI context overview、main PRD 和 module appendices

#### 场景: 用户带前端 URL 执行 generate-prd-from-code
- **当** 用户提供运行中的前端 URL
- **则** 系统捕获所有可见的叶子菜单项和页面结构
- **且** 追踪网络请求到后端路由
- **且** 将前端证据与代码库事实合并，生成增强版 PRD

### 需求: 共享引擎复用

系统应复用 `../../core/scripts/` 目录下的共享脚本，不做修改。

#### 场景: 技术栈检测
- **当** 用户启动 PRD 生成
- **则** 系统调用 `../../core/scripts/detect_stack.py` 识别代码库的技术栈
- **且** 使用检测结果指导后续分析

#### 场景: 代码库事实提取
- **当** 技术栈检测完成
- **则** 系统调用 `../../core/scripts/extract_repo_facts.py` 提取控制器、服务、实体和文档
- **且** 输出结构化的 JSON 事实数据供 PRD 生成使用

#### 场景: 证据合并
- **当** 代码库事实（及可选的前端证据）就绪
- **则** 系统调用 `../../core/scripts/merge_evidence.py` 生成最终 PRD 文档
- **且** 默认使用中文输出，除非另有指定

### 需求: 输出契约合规

系统应生成符合 `core/references/prd-template.md` 定义的输出契约的 PRD 文档。

#### 场景: 主 PRD 生成
- **当** PRD 生成完成
- **则** 主 PRD 包含所有必需章节：文档说明、产品定位、目标角色、顶层导航与信息架构、初始化覆盖检查、模块总览、核心业务对象、核心业务流程、关键状态与业务规则、关键状态机图、前后端闭环说明、页面规格索引、假设与待确认

#### 场景: 模块附录生成
- **当** 分析过程中识别到模块
- **则** 系统为该模块生成附录，包含章节：模块概览、覆盖页面、页面规格、模块业务规则、后端支撑索引、状态与流转、异常处理场景、与其他模块关系、待确认项

### 需求: 证据规则合规

系统应遵循 `core/references/evidence-rules.md` 定义的证据规则。

#### 场景: 前端证据优先
- **当** 提供了前端 URL
- **则** 系统以页面事实作为主要产品证据
- **且** 使用代码和文档补充隐藏的业务规则

#### 场景: 推断透明化
- **当** 系统无法完全验证某个结论
- **则** 该结论在"假设与待确认"章节中明确标注
- **且** 记录推断来源（UI、代码或文档）
